## 🎯 Идея и ценность

### Проблема
Родители хотят, чтобы дети учились, но не знают, как мотивировать без скуки и зубрёжки. Современные образовательные приложения либо перегружены интерфейсом, либо не вовлекают ребёнка в реальную активность.

### Решение
**SmartDino** — обучающая игра с **фиджитал-механикой**:  
- **Цифровая часть** — задания на экране, система очков, магазин.  
- **Физическая часть** — ребёнок лепит из пластилина, рисует, мастерит и отправляет фото родителю через приложение.

### Почему это круто?
- 🔐 **Максимальная простота** — минимум текста, понятные кнопки, акцентные цвета.  
- 👨‍👩‍👧 **Вовлечение родителей** — проверка через PIN-код, подтверждение заданий.  
- 🎮 **Поощрение** — очки + магазин (ребёнок «покупает» реальные призы).  
- 📱 **Кроссплатформенность** — работает как веб-сайт, скоро станет мобильным приложением.  
- 🦕 **Дружелюбный дизайн** — без лишней сложности, только то, что нужно ребёнку.

## 🎮 Сценарий игры (пользовательский путь)

### 👶 Ребёнок (сын)

1. **Регистрация / вход**  
   - Родитель регистрирует ребёнка: указывает email, пароль, придумывает PIN‑код (будет использоваться для входа в родительскую панель).  
   - После регистрации ребёнок автоматически попадает на главный экран.

2. **Главный экран ребёнка**  
   - **Статистика дня** – сколько очков заработано сегодня.  
   - **Дневная серия (streak)** – сколько дней подряд заходил в игру; бонусные очки за ежедневный вход.  
   - **4 основные кнопки:**  
     - `Задания` – список доступных заданий (авто и ручные).  
     - `Ачивки` – полученные достижения (пока только UI‑заглушки).  
     - `Магазин` – покупка реальных призов за очки.  
     - `Профиль` – никнейм, общее количество очков, кнопка перехода к родителю.

3. **Задания (10 типов)**  
   - **Автоматические** (проверяются системой) – например, математический пример: `2+2=`, ребёнок вводит ответ → сразу получает результат «Верно / Неверно» и очки.  
   - **Ручные (фиджитал)** – требуют выполнения в реальном мире: слепить из пластилина, нарисовать, сделать поделку.  
     - Ребёнок выполняет задание **физически**, фотографирует или снимает видео.  
     - Доказательство отправляет **родителю** любым удобным способом (Telegram, WhatsApp, ВК, лично).  
     - В приложении ребёнок нажимает «Отправить на проверку» → создаётся заявка со статусом `pending`.  
   - После отправки ручного задания ребёнок может продолжать играть, пока родитель не проверит.

4. **Профиль и переход к родителю**  
   - В профиле отображается никнейм.  
   - Кнопка **«Я — родитель»** → запрос PIN‑кода (того, что был задан при регистрации).  
   - При правильном PIN‑коде ребёнок (или сам родитель) переходит в родительскую панель.

### 👨‍👩‍👧 Родитель

1. **Родительская панель** (доступна только после ввода PIN‑кода)  
   - **Список заданий на проверку:**  
     - Видны все ручные задания, ожидающие подтверждения.  
     - Кнопки **«Одобрить»** (+50 очков ребёнку) или **«Отклонить»** (без начисления очков).  
   - **Магазин родителя** – не для покупок, а для контроля покупок ребёнка:  
     - Список товаров, которые можно купить за очки (например, «Пицца», «Сникерс», «Poem»).  
     - Логика: ребёнок копит очки, «покупает» пиццу → родитель **обязан вечером купить настоящую пиццу** (или другой приз).  
   - **Статистика ребёнка за день**: сколько заданий выполнено, сколько очков заработано.  
   - **Профиль родителя** – кнопка «Выйти», возврат на главный экран ребёнка.

2. **Обработка заявки**  
   - Родитель видит уведомление (при следующем входе в родительскую панель) о новом ручном задании.  
   - Проверяет доказательство (фото/видео, присланное в мессенджер или загруженное в приложение).  
   - Нажимает **«Одобрить»** → очки ребёнка увеличиваются, задание переходит в статус `approved`.  
   - Ребёнок может сразу потратить полученные очки в магазине.

### 🛍️ Магазин и покупки

- **Магазин ребёнка**  
  - Список товаров (5 стартовых): Snickers, Pizza, Juice, Poem, Gomu Gomu No Mi.  
  - Цена в очках. Если очков достаточно → кнопка «Купить» активна.  
  - После покупки очки списываются, товар попадает в инвентарь (базовая реализация).  
  - *Смысл:* ребёнок реально получает приз от родителя (родитель отслеживает покупки в своей панели).

- **Магазин родителя** (только просмотр)  
  - Показывает, какие товары и когда купил ребёнок.  
  - Помогает родителю выполнить «обязательства» (например, купить пиццу).

### 🔁 Цикл мотивации

1. Ребёнок выполняет авто‑задание → получает очки.  
2. Выполняет ручное задание → присылает доказательство родителю → получает очки после одобрения.  
3. Копит очки → покупает реальный приз в магазине.  
4. Родитель видит покупку → дарит приз.  
5. Ребёнок снова хочет копить → вовлечение в обучение через игру.

---

> **Этот сценарий полностью реализован в связке Unity ↔ FastAPI.**  
> Ручные задания с одобрением родителя работают через эндпоинты `/api/task/complete` (с `task_type=manual`) и `/api/parent/review`.

## Компоненты взаимодействия

- **ApiClient** → отправляет REST-запросы (GET/POST) с JWT-токеном в заголовке.
- **UserSession** → синглтон, хранит `nickname`, `score`, `token`, `parent_pin`.
- **TaskService** → мост между UI и API; вызывает `/task/complete`, `/shop/buy` и т.д.
- **ParentDashboard** → загружает задания на проверку (`/parent/pending-tasks`) и отправляет ревью.
- **CanvasSorter** → переключает панели (Login, Register, Student, Shop, Profile).

### Синхронизация данных

- **Pull-модель** — обновление при открытии страницы (`RefreshList`).
- **Локальная передача** — мгновенное отображение заданий у родителя (без ожидания БД).

### 1. Клонирование репозитория

```bash
git clone <url-вашего-гита>
cd smartdino
```
## 2. Запуск бэкенда (FastAPI)
```bash
cd backend
pip install -r requirements.txt
```
Создайте файл .env или задайте переменные окружения:

```ini
SECRET_KEY=mysecretkey
ADMIN_TOKEN=supersecretadmintoken
DATABASE_URL=postgresql://...
```
Запуск сервера:

```bash
uvicorn main:app --reload --host 0.0.0.0 --port 8000
Swagger документация: http://localhost:8000/docs
```

## 3. Настройка Unity
Откройте проект в Unity Hub (Unity 2022.3.62f3)

В сцене найдите объект GameNetworkManager (или ApiClient)

Укажите URL сервера:

Локально: http://localhost:8000/api

Для Android: http://192.168.x.x:8000/api (IP вашего компьютера)

## 4. Сборка проекта
WebGL (текущая демо-версия): File → Build Settings → WebGL → Switch Platform → Build

Windows / macOS: выберите платформу → Build

5. Тестовый вход
Зарегистрируйте нового ребёнка (email + пароль + PIN) через /api/register

Или используйте тестового пользователя, созданного в Swagger/Postman
